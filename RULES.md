# RULES

These rules govern all agent work in this repository.

---

## NON-NEGOTIABLE RULES

1. Never fabricate facts.
2. Never fake chronology.
3. Never invent emotional reactions.
4. Never generate fictional dialogue.
5. Never convert uncertainty into certainty.
6. Never overwrite user truth.
7. Never remove contradictions silently.
8. Never summarize away important emotional context.
9. Never write as if an inferred event is confirmed.
10. Never present polished narrative as source material.
11. Never use therapeutic framing unless the user explicitly asks for it.
12. Never replace complexity with a cleaner story.
13. Never fabricate locations, dates, people, events, motives, or consequences.
14. Never fill gaps using imagination.
15. Never make prose more dramatic at the cost of factual precision.

---

# INVESTIGATION RULES

The agent must:
- identify timeline gaps
- investigate unclear periods
- ask clarification questions
- revisit earlier inconsistencies
- preserve conflicting versions if unresolved
- keep evidence and interpretation separate
- prefer direct user confirmation over speculation
- actively cross-reference new memories against existing records
- update working memory whenever a gap, contradiction, or open question appears
- keep a current session record of what changed

Every investigation should produce at least one of:
- a clarified timeline placement
- an unresolved question
- a contradiction record
- a new incident record
- an update to an existing record
- a note explaining why the memory cannot yet be placed
- a next-question list for the user

---

# MEMORY RULES

Human memory is imperfect.

The agent should:
- tolerate fragmented memories
- preserve uncertainty honestly
- avoid forcing false structure
- maintain raw notes separately
- avoid treating vividness as proof
- avoid treating vagueness as falsehood
- ask for anchors instead of pressuring exact dates
- separate the user's assumption from the user's memory when both appear
- preserve nonlinear input without forcing a false sequence

Useful anchors include:
- school year
- age
- grade
- city or neighborhood
- device used
- job or role
- friend group
- relationship status
- platform or account
- major family event
- major public event
- before or after another confirmed event

---

# STYLE RULES

Writing style:
- documentary
- archival
- memoir nonfiction
- emotionally grounded
- precise
- human
- restrained
- readable enough for autobiography without becoming fiction

Avoid:
- exaggerated drama
- fake cinematic narration
- motivational tone
- therapy language
- sentimental overstatement
- judgmental labeling
- overexplaining obvious feelings
- invented literary scenes
- implied certainty created by smooth prose

The best style is clear enough to preserve memory and careful enough to be trusted later.

The agent may use novelist-level craft only for nonfiction presentation: structure, pacing, transitions, paragraph flow, and emotional articulation grounded in source material.

---

# DOCUMENTATION RULES

Every major event should include:
- estimated date
- people involved
- place
- emotional impact
- consequences
- uncertainty level
- source notes
- related records
- reliability or source-strength assessment
- open questions

Every archive record should:
- have a stable title
- use a consistent ID when applicable
- preserve uncertainty labels
- link to related records when known
- separate summary from narrative
- separate facts from interpretation
- preserve source references
- include revision notes for major changes

---

# AGENT PRIORITIES

Priority order:

1. truth
2. memory preservation
3. chronology
4. nuance
5. readability

When these priorities conflict, the earlier priority wins.

---

# PRIVACY RULES

This archive may contain sensitive personal information.

The agent must:
- avoid unnecessary exposure of private names in generated summaries
- avoid exporting sensitive material without explicit instruction
- preserve the user's wording in raw notes
- mark information about other people carefully
- distinguish personal memory from verified fact

Do not treat private memory as public material.

---

# REVISION RULES

When revising an existing record:

1. Preserve prior uncertainty unless new evidence changes it.
2. Do not delete meaningful emotional detail.
3. Move superseded material to a revision note if it matters.
4. Update cross-references when filenames or IDs change.
5. Add unresolved issues to working memory.

Major factual revisions should include a short "Revision Note" section explaining what changed and why.

---

# OUTPUT SYSTEM RULES

The archive should be able to support:
- Markdown documents
- JSON exports
- YAML metadata
- website-ready content
- personal wiki pages
- timeline views
- autobiography drafts

Markdown is the default working format.

Structured exports must never add facts that are not present in the Markdown source records.
